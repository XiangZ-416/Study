# 守护线程及线程的优先级

## 一、守护线程
    1.守护线程：为其他线程服务的线程。在JVM中，所有非守护线程都执行完毕后，无论有没有守护线程，虚拟机都会自动退出。
    2.守护线程的作用：将守护线程用于后台支持任务，比如垃圾回收、释放未使用对象的内存、从缓存中删除不需要的条目。
    注意：（1）守护线程不能持有需要关闭的资源（如打开文件等）。
          （2）守护线程必须在非守护线程start之前设置为true才能起作用：线程名.setDaemon(true); 
          （3）守护线程中的守护线程也会随着非守护线程执行完毕而结束
          （4）守护线程中的非守护线程也不会随着非守护线程执行完毕而结束
          
## 二、线程的优先级
    新建一个线程，默认优先级是5。最高可以设置为10，最低为1。
    方法：线程名.setPriority(1 ~ 10);
    注意：执行结果与设置的优先级不一定相符。
